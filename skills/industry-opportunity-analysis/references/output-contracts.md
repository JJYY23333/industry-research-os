# Output Contracts

Use this file when creating a structured workspace or Obsidian-ready deliverables.

## Workspace Structure

```text
<industry-opportunity-os>/
├── README.md
├── 00-scope.md
├── 01-industry-overview.md
├── 02-user-pain-points/
├── 03-competitors/
├── 04-products-offers/
├── 05-keywords-content/
├── 06-business-models/
├── 07-opportunity-cards/
├── 08-scorecard.md
├── 09-validation-plan.md
├── 10-monitoring/
└── sources.md
```

## Minimum First Pass

Create at least:

- `README.md`: how to use and update the workspace.
- `00-scope.md`: industry, region, target user, decision goal, time range, assumptions.
- `01-industry-overview.md`: value chain, customers, channels, top players, constraints.
- `02-user-pain-points/pain-point-map.md`: repeated pains, source snippets, severity, current alternatives.
- `03-competitors/competitor-map.md`: 5-10 players, positioning, products, pricing, channels, weaknesses.
- `04-products-offers/offer-map.md`: categories, price bands, features, review signals, repeat drivers.
- `05-keywords-content/content-opportunity-map.md`: keywords, content types, platform fit, buyer intent.
- `06-business-models/model-map.md`: revenue models, margin logic, CAC/LTV clues, risks.
- `07-opportunity-cards/`: one file per top opportunity.
- `08-scorecard.md`: ranked table with scores and confidence.
- `09-validation-plan.md`: 7/14/30-day validation roadmap.
- `10-monitoring/weekly-template.md`: recurring intelligence template.
- `sources.md`: links, source grade, claim supported.
